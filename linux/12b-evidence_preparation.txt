For analysis of the Windows disk images, we recommend mounting the images with Arsenal Image Mounter and performing a manual examination using the tools of your choice.
For analysis of the Windows memory images, we recommend using Volatility 3 and MemProcFS.

Analysis of the Linux disk and memory images requires a few additional steps. Detailed instructions are provided below.

Mounting the Linux Disk Images (Ubuntu 24.04)
---------------------------------------------

Note: CHI-L-CONF-01 uses the XFS file system and cannot be mounted in WSL 2. Use an Ubuntu 24.04 host system or virtual machine (VM) instead.

CHI-L-CONF-01.raw:

sudo mkdir /mnt/image1 /mnt/image2
sudo apt install lvm2 -y
sudo losetup -f -P /path/to/CHI-L-CONF-01.raw
sudo pvscan --cache
sudo vgchange -ay
sudo vgdisplay
sudo mount -o ro,norecovery -t xfs /dev/mapper/almalinux-root /mnt/image1

CHI-L-WEB-04.raw:

sudo losetup -f -P /path/to/CHI-L-WEB-04.raw
sudo pvscan --cache
sudo vgchange -ay
sudo vgdisplay
sudo mount -o ro,loop -t ext4 /dev/mapper/ubuntu--vg-ubuntu--lv /mnt/image2

Analyzing the Linux Memory Images
---------------------------------

python3 vol.py -f /mnt/c/path/to/CHI-L-CONF-01.mem -u https://raw.githubusercontent.com/Abyss-W4tcher/volatility3-symbols/refs/heads/master/banners/banners.json linux.pslist
python3 vol.py -f /mnt/c/path/to/CHI-L-WEB-04.mem -u https://raw.githubusercontent.com/Abyss-W4tcher/volatility3-symbols/refs/heads/master/banners/banners.json linux.pslist

This will automatically download the correct symbols for each image. You can omit the -u URL on subsequent plugin runs.
